## 工作原理

高拍仪拍摄手机屏幕，PC端 OCR 识别题目，ADB 模拟点击自动作答。

全程在手机外部完成——摄像头是独立硬件，识别在电脑上运行，点击模拟真实手指触控。手机端无法感知任何自动化行为，对它而言每一次触摸都是正常的物理输入。无论考试平台如何检测录屏、截屏、应用钩子、后台进程或无障碍服务，均无法察觉本工具的存在。

## 功能

- 本地离线识别，无需联网
- 高拍仪拍摄屏幕，OCR 自动识别题目
- 模糊匹配算法，容错率高
- 识别后自动点击选项，全程无需手动操作
- Excel 格式题库，方便编辑维护
- 自动检测摄像头，支持切换
- 首次识别锁定坐标系，后续复用
- 参数自动保存，下次启动无需重新设置
- 选项乱序兼容，按内容匹配
- 判断题自动识别圆圈标识和文字选项
- 可选多次识别，截3次取众数，更准但慢

## 硬件要求

- Windows 10/11
- 高拍仪（推荐）或 USB 摄像头
- 小米手机（开启 USB 调试）

## 题库格式

Excel 文件，三列：题目 | 选项 | 答案

| 题目 | 选项 | 答案 |
|------|------|------|
| 题目内容 | A. 选项1 \| B. 选项2 \| C. 选项3 \| D. 选项4 | A |

- 选项用 `|` 分隔
- 单选填字母 A/B/C/D，多选填多个字母，判断题填 A 或 B
- 判断题选项为"正确/错误"、"对/错"、"√/×"时自动识别

## 使用方法

1. 下载 `ExamTerminator.exe` 和 `question_bank_template.xlsx`（放同一目录）
2. 按实际题目编辑题库
3. 连接高拍仪与小米手机，手机开启 USB 调试
4. 双击运行，加载题库，点击"开始答题"

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 题目匹配 | 0.5 | 题目相似度阈值，越高越严格 |
| 选项匹配 | 0 | 选项相似度阈值，越高越严格 |
| 题前冷却 | 3.0 | 答题间隔（秒），考试要求至少3秒 |
| 选项偏移 | 200 | 选项点击 Y 轴偏移 |
| 按钮偏移 | 200 | 下一题按钮 Y 轴偏移 |
| 多次识别 | 关 | 截3次取众数，更准但慢 |

## 从源码运行

```bash
git clone https://github.com/cobetckoe/ExamTerminator.git
cd ExamTerminator
pip install -r requirements.txt
cd src
python exam_terminator.py
```

**注意**：从源码运行需要确保ADB工具已安装在`src/platform-tools/`目录下。
